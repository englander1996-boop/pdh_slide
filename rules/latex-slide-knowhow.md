## ⭐ 最重要：制作ワークフロー（推測で詰めない）

数値や寸法を当て推量で何度も直すと時間を溶かす。**1ループ = 書く → 2回コンパイル → 画像化 → 目視**を徹底する。

```bash
# 該当ページだけ高速プレビュー（N はページ番号）
lualatex -interaction=nonstopmode main.tex      # 2回
pdftoppm -png -r 120 -f N -l N main.pdf preview  # 1ページだけPNG化して目視
```

- はみ出し・重なりは**1要素ずつ**直す（複数同時に動かすと原因が切り分けられない）。
- 「たぶん収まる」で進めない。**必ず画像で確認**してから次へ。

## A. 環境・ビルド

| 症状 | 原因 | 対策 |
|---|---|---|
| 日本語が出ない／フォントエラー | `pdflatex` で luatexja を使った | **必ず `lualatex`**（または `latexmk -lualatex`）。pdflatex 不可 |
| `This package requires Lua(HB)(La)TeX` ／ `For rule 'latex', no output was made` | エディタ（VS Code / LaTeX Workshop 等）がデフォルトの `latex`(pdfTeX) ルールでビルドした。luatexja は LuaLaTeX 以外を拒否する | プロジェクト直下に **`.latexmkrc`** を置き `$pdf_mode = 4;`(lualatex) を固定。さらに `main.tex` 先頭に **`% !TEX program = lualatex`** を入れる。これでエディタ・CLI どちらも LuaLaTeX になり、`latexmk` 単体（`-lualatex` 無し）で通る |
| 目次・ページ番号・グラフ参照がズレる | 1回しかコンパイルしていない | **2回コンパイル**（pgfplots や toc は 2 パス必要） |
| 作業ディレクトリが散らかる | `.aux .log .nav .snm .toc` が残る | ビルド後に掃除、または `.gitignore` に登録（`main.pdf` も成果物として除外可） |

## B. テーマ・互換性

| 症状 | 原因 | 対策 |
|---|---|---|
| `[standout]` 等で `Undefined` エラー | **テーマ固有のframeオプション**を別テーマで使った（Metropolis 専用機能を default テーマで使用） | テーマ変更時はテーマ依存の機能を洗い出して置換。汎用機能だけ使う |
| 色やコマンドの二重定義・不整合 | スライド側で `\definecolor` や `\newcommand` を再定義 | **色・共通コマンドは `style/theme.tex` に集約**。スライドは参照のみ（例: `\bluearrow` `\hilite` `\paramtable`） |

## C. レイアウト・はみ出し（最頻出）

- **4:3 フレームの縦は有限**。ヘッダーバー＋パラメータ表＋グラフ＋結論を詰め込むと簡単に溢れて**下端が切れる**。先に「縦の予算」をざっくり見積もる。
- グラフ高さの目安：**ヘッダーバー付き・2カラム構成なら 1プロット ≈ 2.3〜2.4cm**。これ以上だと下部の結論が切れやすい。
- **pgfplots の落とし穴 `scale only axis`**：`width=` は**プロット描画領域だけ**の幅。軸ラベル・目盛りラベルの幅は**別途加算**される。
  - → カラム幅（4:3で `0.48\textwidth` ≈ 5.2cm）に対し、`width` は**両側の軸ラベルぶん（約1.5〜2cm）を引いた値**にする。今回は `width=3.5cm` で収まった（`4.3cm` だと隣と衝突）。
- **二軸グラフを2カラムで並べると中央の継ぎ目で衝突**（指摘事例）：
  - 「左プロットの**右**軸ラベル(IPA選択率)」と「右プロットの**左**軸ラベル(生成量)」が継ぎ目で重なる。
  - → プロット幅を詰める／カラム間に余白を作る／必要なら内側の軸ラベルを短縮。

## D. 日本語テキストの折り返し（指摘事例）

| 症状 | 原因 | 対策 |
|---|---|---|
| 「水/プロピレン流量比を **1:1**」で改行、「に固定」だけ孤立 | 狭いカラムに長い和文を入れ、不自然な位置で改行 | ① 結論文は**カラム外の全幅 `minipage`** に出す ② フォントを1段下げる（`\small`→`\footnotesize`）で1行化 ③ 割れて困る語は `\mbox{...}` で**分割禁止**（例: `\mbox{$1\!:\!1$に固定}`） |

- 比・単位・記号＋語句（`1:1に固定`, `160°Cに保持`）は**ひとかたまり**として `\mbox`。

## E. TikZ / pgfplots 構文

| 症状 | 原因 | 対策 |
|---|---|---|
| `I do not know the key '/tikz/upper left'` | グラデ指定のキー名が無効 | 斜めグラデは `\shade[top color=A, bottom color=B, shading angle=20]`。`upper left=` 等は存在しない |
| 矢印 `-{Stealth}` が出ない／エラー | tikzライブラリ未読込 | プリアンブルで `\usetikzlibrary{arrows.meta}` |

## F. ページ番号

- 表紙は `\begin{frame}[plain]`（ヘッダー・番号なし）。**表紙直後に `\setcounter{framenumber}{0}`** で次スライドを 1 にする。
- 単体テストで「15ページ目」の見た目を再現したいだけなら `\setcounter{framenumber}{14}` で偽装可。**本番ファイルからは必ず削除**。

- 2026-06-08 追記。上の表を帯から下げたいが下端の図が `[t]` で詰まっている板は、**上 vspace（要素を下げる）と要素間 vspace（下端を回収）を連動**させて「下げる量・下端 Overfull ゼロ・重なり回避」の3点を同時に満たす。間 vspace を負に振りすぎると次要素（図のタイトル等）が前の表に**重なる**ので、画像で必ず重なり確認。微調整は両 vspace を 1pt ずつ逆向きに動かすと下端を保ったまま位置だけ変えられる。画像内タイトル（[ ] 等）の体裁変更は .tex でなく生成スクリプト側（make_*.py）を直して再生成。

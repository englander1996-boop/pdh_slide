# スライド作成ルール集（LaTeX / Beamer 版）

> 元の Marp 版（`Z:\powerpoint_for_processdesign\markdowns\rules\slide-creation-rules.md`）を
> LuaLaTeX + Beamer 向けに改訂したもの。デザインの見た目（濃青ヘッダーバー・専用表紙）は踏襲しつつ、
> LaTeX で自動化できる部分は自動化した。**[改善]** マークは Marp 版からの変更点。

## 目的
- ゼミ／学会用スライドを安定して作るための表記・レイアウト・運用ルールを LaTeX 前提で明文化する。

## 適用範囲
- 対象: `Y:\pdh_slide` 以下の Beamer スライド一式。
- 対象者: スライド作成者、レビュワー。

## 用語
- 「板 / スライド」: Beamer の 1 フレーム（`\begin{frame}...\end{frame}`）。表紙＝タイトルフレーム。

## ビルド方法
- エンジンは **LuaLaTeX**（和文 luatexja のため pdflatex 不可）。
- コマンド: `lualatex main.tex` を **2回**（目次・参照解決のため）。または `latexmk -lualatex main.tex`。
- 出力は `main.pdf`。

## ファイル構成・メタデータ
- `main.tex`: プリアンブルと読み込みのみ。タイトル情報（`\title` 等）と `\input` 行の順序がスライド順を決める。
- `style/theme.tex`: テーマ定義（配色・ヘッダーバー・表紙コマンド）。**見た目の変更はここに集約**する。
- `slides/NN-topic.tex`: スライド本体を **1ファイル＝1スライド** で置く。**[改善]** Marp は1ファイルに全板だったが、LaTeX では差し替え・並べ替えを `main.tex` の `\input` 行編集だけで行える。
- `figures/`: 画像。`\graphicspath` 済みなのでファイル名のみで参照可。
- 内部ソースIDのトラッキングは各スライドファイル冒頭の `% source_memo: {...}` コメントに残す（出力されない）。

## 命名規則
- ファイル名は英数小文字とハイフン、先頭に2桁連番（例: `01-background.tex`, `04-results-table.tex`）。
- 連番はスライド順の目安。途中挿入は `02b-...` 等でも可だが、安定後はリネーム整理する。

## レイアウト共通ルール
- スライド比率は **4:3**（`\documentclass[aspectratio=43]`）。
- 全本文スライド上部に**恒常ヘッダーバー**（濃青 `#0f5a7a`／白文字／太字）。左にセクション名（`\insertsectionhead`）、右にページ番号。
  - ロゴを入れる場合は `theme.tex` の `\HeaderLogo` を `\renewcommand` で画像指定。
- **[改善] ページ番号は自動**（`\insertframenumber`）。Marp 版の「手動採番・自動カウンタ禁止」は不要 — LaTeX は挿入・削除・分割時に**自動で再採番**されるため手作業が消える。
  - 表紙はページ番号なし（`[plain]` フレーム）。表紙直後に `\setcounter{framenumber}{0}` を置き、次スライド（目次）を 1 とする。
- フォントサイズ目安（`theme.tex` で調整）: スライド見出し `\large` 太字、本文は既定（11pt ベース）、見出し色は濃青。
- 配色定義は `theme.tex` の `HeaderBlue` / `DateGray` / `TitleBG` を編集すれば全体に反映。

## 表紙ルール
- 表紙は `main.tex` 内で `\seminartitlepage`（`theme.tex` 定義）を呼ぶだけ。毎回同じ構成・見た目に固定される。
- 表紙では通常のヘッダーバー・ページ番号を表示しない。
- 構成（固定・上から）:
  1. **アクセントライン**: 濃青 `#0f5a7a` の横線（左上、約 16mm×1.1mm）
  2. **メインタイトル**（`\title`、約26pt 太字）
  3. **副題**（`\subtitle`、約16pt 濃青 太字）
  4. **著者名**（`\author`、約13pt。複数行・肩付き可）
  5. **日付**（`\date`、約11pt グレー `#334b5e`）
- **変更可能なのは原則 `\title` / `\subtitle` / `\date`**。必要時のみ `\author`。位置・余白・色・サイズは `theme.tex` 側のみで変更し、個別スライドでは触らない。

## 文字・表記ルール
- **[改善] 数式・数値はネイティブ LaTeX**。インライン `$...$`、ディスプレイ `\begin{equation}` 等。Marp のような「数式レンダリングを有効化」設定は不要。
- 数値・単位も一貫して数式モードで（例: `$404$`, `$-42.05$`, `$12.0\,\mathrm{ms}$`）。
- 和欧混在は読みやすさ優先。luatexja が和欧間スペースを自動調整する。

## 引用・出典・脚注
- スライド内の引用ラベルは学術スタイルの番号 `[1]` のみ表示。内部ソースID（例 `004`）は**絶対に可視化しない**（`% source_memo` コメントへ）。
- 参考文献は末尾（`slides/99-references.tex`）に番号付きでまとめる。
- **[改善] 採番の選択肢**:
  - 現状は `thebibliography` の手動リスト（軽量）。
  - 出典が増える場合は **biblatex + `\cite{}`** を推奨。登場順の連番・再採番・参考文献生成が**自動**化され、Marp 版の「挿入削除時に手動再採番」が不要になる。

## 画像・図・表
- 画像は `figures/` に置き、`\includegraphics[width=...]{name}` で参照（パスはファイル名のみ）。
- 表は `booktabs`（`\toprule`/`\midrule`/`\bottomrule`）で簡潔に。罫線多用を避ける。
- 図表のキャプションは短く、注記は最小限に。

## オーバーフロー時のルール
1. まず本文を `\small` 等で少し下げる。
2. 次に `\setlength` で行間・項目間を詰める、または箇条書きを整理。
3. それでも収まらなければ**スライドを分割**（`slides/` に新ファイルを追加し `main.tex` に `\input`）。**[改善] 分割してもページ番号・引用番号は自動で追従**するため再採番作業は不要。
- 長い表・図が多い場合は分割を優先（可読性重視）。
- 縦に溢れがちな参考文献等は `\begin{frame}[allowframebreaks]` で自動分割可。
- **[改善] 過密検知**: Marp の `check-slide-density.mjs` は不要。LaTeX では `\documentclass` に `[shrink]` を付けた frame や、コンパイル時の `Overfull \vbox` 警告（`main.log`）で過密を検知する。

## スライド内容ルール
- **1スライド = 1トピック**。
- 箇条書きは短く、冗長表現を避ける。
- セクション見出しは明確に。重複文言を避ける（ヘッダーバーに `\insertsectionhead` が出るため、`\section{}` 名は簡潔に）。

## レビュー・運用ワークフロー
- 変更は最小差分。スライド順や引用に影響する場合は説明に明記。
- 構成の大きな再編は事前合意を取る。
- バージョン管理する場合、`main.pdf` や `*.aux`/`*.log` 等の中間ファイルは `.gitignore` 推奨。

## エクスポートとロールバック
- 最終出力 `main.pdf` を配布物として扱う。配布版は `powerpoints/`（または `dist/`）へコピー。
- 戻す場合は Git revert/checkout（リポジトリ化している場合）。

## 互換性・参照先
- 見た目の元仕様は Marp 版ルール（`Z:\...\slide-creation-rules.md`）。色・レイアウトの意図はそちらと整合させる。
- 本ファイルは LaTeX 実装（`style/theme.tex`）と必ず整合させる。

## 例（短縮）
- 数値: `$404$`。
- 引用: 初出 `[1]`、次 `[2]`（biblatex 採用時は `\cite{}` で自動）。
- 内部ソース: スライド先頭に `% source_memo: {"id":"004","note":"内部参照用"}`。

---

**Marp 版からの主な改善まとめ**
1. ページ番号: 手動 → **自動**（挿入・分割で再採番不要）
2. ファイル: 1ファイル全板 → **1スライド1ファイル**（並べ替えが `\input` 編集だけ）
3. 数式: レンダラ設定不要の**ネイティブ LaTeX**
4. 引用: biblatex 採用で**自動採番・自動文献生成**が可能
5. 過密検知: 専用スクリプト不要、**コンパイル警告**で検知
